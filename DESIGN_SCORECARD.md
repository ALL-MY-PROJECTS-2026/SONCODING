## 🎯 트렌드 리서치 메모 (2026)
- **스크롤 기반 애니메이션**: 순수 CSS `animation-timeline: view()`로 JS 없이 구현 가능 → 등장 리빌·패럴랙스
- **모션 디자인 기본화**: 페이지 전환·호버·로딩 시퀀스가 표준 기대치
- **마이크로 인터랙션**: 떠다니는 요소, 부드러운 피드백
- **키네틱 타이포그래피**: 글자 단위 등장, 호버 변형
- **3D/뎁스 (WebGL)**: 고급 사이트에서 활용 (후순위, 무게 큼)
- 원칙: 화려함보다 **깔끔함 + 성능 + 접근성** 유지

## 📝 회차 로그

### 회차 1 — 모션 기반 (Motion foundation)
- 순수 CSS 스크롤 리빌(`.reveal`) 도입 — `animation-timeline: view()` 기반, 미지원/reduced-motion 시 그대로 표시(우아한 폴백)
- 히어로 배경 글로우에 부드러운 플로팅 애니메이션
- 홈·내부 페이지 주요 섹션에 리빌 적용
- 영향: 항목 2, 6

### 회차 2 — 키네틱 숫자 & 스크롤 진행바
- 통계 숫자 **카운트업 애니메이션**(뷰포트 진입 시 0→목표, easeOutCubic) — IntersectionObserver, reduced-motion 시 최종값 즉시 표시, SSR엔 실제 숫자 렌더(SEO 안전)
- 상단 **스크롤 진행바** — 순수 CSS `animation-timeline: scroll(root)`, 미지원 시 미표시(우아한 폴백)
- 영향: 항목 2 (7→8), 항목 4 (6→7), 항목 6 (6→7)

### 회차 3 — 히어로 진입 & 키네틱 그라디언트 헤드라인
- 히어로 요소(배지·제목·설명·버튼) **스태거 진입 애니메이션**(로드 시 rise-in, 순수 CSS, reduced-motion 존중)
- 제목에 **키네틱 그라디언트**(슬레이트→블루→인디고 스윕, background-clip:text) — 미지원 시 솔리드 폴백
- 진입+그라디언트 애니메이션 shorthand 충돌 해결(결합 규칙)
- 영향: 항목 1 (7→8), 항목 3 (7→8), 항목 4 (7→8)

### 회차 4 — 인터랙티브 배너 배경 (오로라 + three.js 파티클)
- **오로라/메시** 배경(순수 CSS, 저작권 무관): 홈 히어로 + 내부 페이지 배너 + About 히어로에 부드럽게 흐르는 컬러 블롭
- **three.js 생성 파티클 배경**(홈 히어로): 블루/시안/인디고/바이올렛 발광 입자 560개(모바일 280개), 가산 블렌딩, 원형 스프라이트
- **마우스 인터랙션**: 배너 위 커서 방향으로 파티클 필드가 패럴랙스 기울임(부드러운 lerp) + 느린 자동 회전
- 성능: three.js는 홈 라우트에만 코드 분할, DPR≤2, 뷰포트 밖이면 애니메이션 정지, reduced-motion 시 정지 프레임, 언마운트 시 완전 정리
- 저작권: 외부 이미지·영상 없음(전부 코드 생성)
- 영향: 항목 2 (8→9), 항목 3 (8→9), 항목 5 (7→8), 항목 8 (9→8, three.js 무게)
